# API Spec

This file describes the MCP tool surface as implemented in the current code.
All field names and types match the Zod schemas in `src/mcp/schemas.ts` and the
response types in `src/domain/contracts/types.ts`.

---

## Global Response Envelope

Every successful tool response is wrapped with `withLimsMarker` in `register-tools.ts`.
All error and success payloads include:

```jsonc
{
  // ... tool-specific fields ...
  "status": "LIMS_ACTIVE",
  "_lims": {
    "provider": "LIMS MCP",
    "inUse": true,
    "tool": "<tool-name>",
    "message": "Currently LIMS MCP is in use. Response generated by LIMS for tool \"<tool>\"."
  }
}
```

Error responses (`isError: true`) contain `{ error: string, message: string }` from `DomainError` or `ValidationError`.

## `capture_generate_locator`

**Purpose:** Open a live web page or parse provided HTML using `CompositePageCaptureAdapter`
(tries `PlaywrightMcpValidatorAdapter` first, falls back to `PlaywrightWebCaptureAdapter`),
then run the full locator-generation pipeline.

### Input

| Field | Type | Required | Notes |
|---|---|---|---|
| `platform` | `"web"` | Yes | Only web is supported here |
| `pageUrl` | `string` | One of ↓ | Live URL to navigate to |
| `html` | `string` | One of ↓ | Raw HTML string |
| `target` | `string` | Optional | Human-readable element description |
| `targetDescriptor` | `TargetDescriptor` | Optional | Structured descriptor (`label`, `role`, `text`, `testId`, `name`, `region`) |
| `runtimeContext` | `RuntimeContext` | Optional | `useCurrentPage`, `retries`, `waitForStable`, `timeoutMs` |
| `captureContext` | `CaptureContext` | Optional | `captureScreenshot`, `captureDOM`, `captureFull`, `captureInteractive` |
| `includeInteractiveElements` | `boolean` | Optional | Capture clickable/interactive elements map |
| `captureFullPage` | `boolean` | Optional | Full-page vs viewport screenshot |

### Output

Normal `GenerateLocatorResponse` fields (see `generate_locator`) **plus**:

| Field | Type | Notes |
|---|---|---|
| `capture.targetDescriptor` | `TargetDescriptor` | Descriptor inferred from live page |
| `capture.interactiveElements` | `InteractiveElement[]` | All interactive elements found |
| `capture.artifact` | `{ ref: string }` | Artifact store key for feedback/healing |
| `capture.dom` | `string` | Raw captured DOM (if DOM capture enabled) |
| `capture.screenshotBase64` | `string` | Screenshot (if screenshot enabled) |

## `generate_locator`

**Purpose:** Generate ranked locators from an existing snapshot (no live browser needed).

### Input

| Field | Type | Required | Notes |
|---|---|---|---|
| `platform` | `"web"\|"android"\|"ios"` | Yes | Drives tier strategy and snippet output |
| `dom` | `string` | One of ↓ | HTML string |
| `xml` | `string` | One of ↓ | UIAutomator / XCUI XML string |
| `screenshot` | `string` | One of ↓ | Base64 image (visual-only path if no DOM) |
| `domFile` | `string` | One of ↓ | File path — resolved and read in MCP layer |
| `xmlFile` | `string` | One of ↓ | File path — resolved and read in MCP layer |
| `screenshotFile` | `string` | One of ↓ | File path — resolved and read in MCP layer |
| `domSnapshots` | `string[]` | Optional | Multiple DOM snapshots for temporal stability |
| `xmlSnapshots` | `string[]` | Optional | Multiple XML snapshots for temporal stability |
| `screenshotSnapshots` | `string[]` | Optional | Multiple screenshots for temporal stability |
| `target` | `string` | Optional | Human-readable element description |
| `targetDescriptor` | `TargetDescriptor` | Optional | Structured descriptor |
| `targetRegion` | `string` | Optional | `"top-right"` etc. for chart/canvas region |
| `runtimeContext` | `RuntimeContext` | Optional | `useCurrentPage`, `retries`, `waitForStable`, `timeoutMs` |

### Output

| Field | Type | Notes |
|---|---|---|
| `bestLocator` | `string` | Highest-confidence locator |
| `confidence` | `number` | Fused confidence `[0, 1]` (dom×0.5 + visual×0.2 + runtime×0.3, renormalised) |
| `strategy` | `string` | Strategy that produced `bestLocator` |
| `fallbacks` | `RankedLocator[]` | Ranked alternatives with individual scores |
| `validation` | `RuntimeValidation` | `{ executed, unique, visible, interactable, success, notes, source }` |
| `automation` | `AutomationSnippets` | Platform code snippets (Playwright, Selenium, Appium, etc.) |
| `metadata` | `object` | `fingerprint`, `framework`, `visualHints`, `temporalStability`, `dynamicTextClassification` |
| `locatorCatalog` | `LocatorCatalog` | `{ total, items:[{ rank, locator, category, source, strategy, score, runtimeSuccess, ... }], rejected }` |
| `explanation` | `string` | Why this locator was chosen |

## `heal_locator`

**Purpose:** Recover a broken locator after a UI change.
First tries the old locator directly; if it no longer uniquely resolves, applies
`SimilarityEngine.bestMatch` (weighted: `tag×0.12 + text×0.28 + attrs×0.35 + hierarchy×0.25`)
then regenerates on the matched element.

### Input

| Field | Type | Required | Notes |
|---|---|---|---|
| `oldLocator` | `string` | Yes | The locator to recover |
| `dom` | `string` | One of ↓ | Latest HTML snapshot |
| `xml` | `string` | One of ↓ | Latest XML snapshot |
| `screenshot` | `string` | One of ↓ | Latest screenshot (base64) |
| `domFile` | `string` | One of ↓ | File path to latest HTML |
| `xmlFile` | `string` | One of ↓ | File path to latest XML |
| `screenshotFile` | `string` | One of ↓ | File path to screenshot |
| `domSnapshots` | `string[]` | Optional | Multiple snapshots for temporal stability |
| `screenshotSnapshots` | `string[]` | Optional | Multiple screenshots |
| `platform` | `"web"\|"android"\|"ios"` | Optional | Defaults to `"web"` |
| `fingerprint` | `object` | Optional | Previously stored element fingerprint |
| `runtimeContext` | `RuntimeContext` | Optional | Runtime validation options |

### Output

| Field | Type | Notes |
|---|---|---|
| `healedLocator` | `string` | Recovered locator (may equal `oldLocator` if still valid) |
| `confidence` | `number` | Fused confidence `[0, 1]` |
| `explanation` | `string[]` | Why each candidate was chosen/rejected |
| `diff` | `object` | `{ oldLocator, healedLocator, changed: boolean }` |
| `validation` | `RuntimeValidation` | Runtime check result on healed locator |
| `fallbacks` | `RankedLocator[]` | Ranked alternatives |
| `locatorCatalog` | `LocatorCatalog` | Full ranked catalog |

## `report_locator_result`

**Purpose:** Record a test pass/fail outcome and optionally trigger self-healing.
Writes to both `LocatorArtifactStore` (per-element JSON) and `LocatorLearningStore` (global log).

### Input

| Field | Type | Required | Notes |
|---|---|---|---|
| `locator` | `string` | Yes | The locator that was tested |
| `status` | `"passed"\|"failed"` | Yes | Outcome |
| `platform` | `"web"` | Yes | Only web feedback supported currently |
| `artifactRef` | `string` | Preferred | Ref from `capture_generate_locator` |
| `pageUrl` | `string` | Optional | Used as fresh DOM source for healing |
| `html` | `string` | Optional | Raw HTML for healing |
| `dom` | `string` | Optional | Parsed DOM for healing |
| `screenshot` | `string` | Optional | Screenshot for healing |
| `target` | `string` | Optional | Element description for re-generation |
| `targetDescriptor` | `TargetDescriptor` | Optional | Structured descriptor |
| `runtimeContext` | `RuntimeContext` | Optional | |
| `captureContext` | `CaptureContext` | Optional | |
| `fingerprint` | `object` | Optional | Previously stored fingerprint |
| `failureMessage` | `string` | Optional | Playwright/test runner error text |

### Output

| Field | Type | Notes |
|---|---|---|
| `learned` | `boolean` | Whether outcome was recorded in learning store |
| `artifact` | `object \| null` | Updated artifact summary |
| `improved` | `object \| null` | If healed: `{ healedLocator, confidence, diff, explanation, validation }` |
| `insights` | `object` | `{ preferredLocators, failedLocators, healedPairs }` from learning store |

## `sync_playwright_framework`

**Purpose:** Create or update Playwright framework files using generated locators.
Uses `mergeGeneratedBlock` — custom code outside `<lims:*>` markers is preserved.

### Input

| Field | Type | Required | Notes |
|---|---|---|---|
| `feature` | `string` | Yes | Feature name (used to derive class names and file names) |
| `locatorBindings` | `PlaywrightLocatorBindingInput[]` | Yes | At least one `{ name, locator?, artifactRef?, testId? }` |
| `language` | `"ts"\|"js"` | Optional | Inferred from existing file extension; defaults to `"ts"` |
| `pageUrl` | `string` | Optional | Inserted into `page.open()` |
| `testCases` | `string[] \| PlaywrightTestCase[]` | Optional | Test descriptions or `{ name, steps }` objects |
| `outputDir` | `string` | Optional | Root output directory |
| `specDir` | `string` | Optional | Sub-directory inside `outputDir` for spec file |
| `pageDir` | `string` | Optional | Sub-directory inside `outputDir` for page file |
| `locatorDir` | `string` | Optional | Sub-directory inside `outputDir` for locator file |

### Files Written

```
{outputDir}/{locatorDir ?? '.'}/{featureBase}.locator.{ts|js}
{outputDir}/{pageDir    ?? '.'}/{featureBase}.page.{ts|js}
{outputDir}/{specDir    ?? '.'}/{featureBase}.spec.{ts|js}
```

### Output

| Field | Type | Notes |
|---|---|---|
| `written` | `string[]` | Absolute paths of written files |
| `locatorNames` | `string[]` | Method names generated in locator file |
| `warnings` | `string[]` | E.g. "Block appended — no existing markers found" |

## `analyze_dom`

**Purpose:** Inspect a DOM/XML snapshot and return intelligence for locator strategy.
Runs `FrameworkDetector`, `AttributeStabilityAnalyzer`, `CanvasElementDetector`, and trading-hint logic.

### Input

| Field | Type | Required | Notes |
|---|---|---|---|
| `dom` | `string` | One of ↓ | HTML string |
| `xml` | `string` | One of ↓ | UIAutomator / XCUI XML |
| `domFile` | `string` | One of ↓ | File path |
| `xmlFile` | `string` | One of ↓ | File path |

### Output

| Field | Type | Notes |
|---|---|---|
| `framework` | `FrameworkProfile` | Detected framework: `react`, `angular`, `vue`, `flutter-web`, `ag-grid`, `unknown`, etc. |
| `mode` | `"html"\|"xml"` | Parsing mode used |
| `recommendedAttributes` | `string[]` | Stable test attributes found in the snapshot |
| `stabilityReport` | `StabilityReport` | Per-attribute stability ratings: `STABLE \| SEMI_STABLE \| UNSTABLE` |
| `hints` | `string[]` | Human-readable strategy hints (e.g. "Use data-testid — found on N elements") |
| `visualHints` | `VisualHints` | `{ hasCanvas, hasSvg, hasWebgl, chartLibrary?, tradingHints? }` |

## `health_check`

**Purpose:** Verify prerequisites and the effective runtime validation mode.
Exposes the result of `detectRuntimeMode` inferred from runtime validator notes.

> **Note:** `playwrightMcpConfigured` in the response currently reflects only whether
> `LIMS_PLAYWRIGHT_MCP_COMMAND` is set. If you use `LIMS_PLAYWRIGHT_MCP_URL` (HTTP mode),
> the MCP adapter is active but `playwrightMcpConfigured` may appear `false` — the
> `runtime.mode` field is the authoritative indicator.

### Input

None.

### Output

| Field | Type | Notes |
|---|---|---|
| `healthy` | `boolean` | `true` if no blocking issues |
| `prerequisites` | `object` | `{ tesseractAvailable, playwrightPackageInstalled, playwrightMcpConfigured }` |
| `runtime` | `object` | `{ mode: "playwright-mcp" \| "http-bridge" \| "local-fallback", detail: string }` |
| `config` | `object` | Active env-config summary |
| `issues` | `string[]` | Blocking problems |
| `warnings` | `string[]` | Non-blocking concerns |

## Scope Note

| Area | Status |
|---|---|
| **Web (Playwright)** | Full: capture, generate, validate, heal, feedback, framework sync |
| **Android (Appium XML)** | Snapshot-based generate/heal only — no live device adapter |
| **iOS (XCUI XML)** | Snapshot-based generate/heal only — no live device adapter |
| **Selenium** | CSS/XPath output compatible; no live Selenium validation adapter |
| **OCR/Vision** | Available when `tesseract` is installed; otherwise gracefully absent |
